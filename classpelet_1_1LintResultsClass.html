Detailed Description

Holds the results of the lint check. Currently lint check will stop when the first error is encountered.

int pelet::LintResultsClass::CharacterPosition

The character offset in which the error ocurred (with regargs to the start of the file). This is 0-based.

UnicodeString pelet::LintResultsClass::Error

A short description of the error; this is generated by the bison parser and is not the most user-friendly but it is exactly what PHP displays; might as well keep it consistant.

std::string pelet::LintResultsClass::File

Path to the file in which the error ocurred. This is what was given to the LintFile() method or ScanFile(std::string, LintResultsClass&) method. For LintString() results this will be the empty string.

int pelet::LintResultsClass::LineNumber

The line in which the error ocurred. This is 1-based.

pelet::ScopeClass pelet::LintResultsClass::Scope

If the parser encountered a syntax error, then this object will be filled with the last known class/method/function where the error occurred.

UnicodeString pelet::LintResultsClass::UnicodeFilename
